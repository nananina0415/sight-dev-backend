# 동방 도어락 회원 명단

동방 도어락(라즈베리파이)이 오프라인(백엔드 연결 불가) 상황에서도 출입을 검증할 수 있도록 회원 명단을 백업하기 위한 API입니다. 도어락 검증에 필요한 **이름(`realname`)과 학번(`number`)만** 반환합니다. SYSTEM API 키 인증을 사용합니다.

> 기존 `GET /manager/users`(listUsers)는 `MANAGER` 권한이고 이메일·전화번호 등 도어락에 불필요한 정보까지 반환하므로, SYSTEM 권한 + 최소 필드 전용으로 본 API를 별도로 둔다.

## API

```
GET /internal/door-lock/members
```

### 요청 헤더

| 이름        |  타입  | 설명           |
| :---------- | :----: | :------------- |
| `x-api-key` | 문자열 | SYSTEM 인증 키 |

### 쿼리 파라미터

(해당 없음)

### 요청 바디

(해당 없음)

### 응답 코드 및 응답 바디

```
200 OK
```

| 이름             |   타입    | 설명                       |
| :--------------- | :-------: | :------------------------- |
| `count`          |   숫자    | 반환된 회원 수             |
| `members`        | 객체 배열 | 회원 명단                  |
| `members.name`   |  문자열   | 회원 실명 (`realname`)     |
| `members.number` |   숫자    | 학번 (`number`)            |

학번(`number`)이 없는 회원은 도어락 출입 검증 대상이 될 수 없으므로 명단에서 제외한다. (출입 검증 기준은 [`door-lock-access.md`](door-lock-access.md)와 동일하게 학번 기준이며, 상태(`status`)별 추가 필터가 필요하면 두 API를 함께 맞춘다.)

### 테스트 케이스

1. `x-api-key` 유효 → 200, 회원 명단(`name`·`number`) 반환
2. 응답에 이메일·전화번호 등 도어락에 불필요한 필드는 포함되지 않는다
3. 학번(`number`)이 없는 회원은 명단에서 제외된다
4. `x-api-key` 불일치 → 401
5. 명단 대상 회원이 없으면 → 200, `count=0` 빈 배열
