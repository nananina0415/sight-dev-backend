# 그룹 활동 일정 삭제

그룹 활동(`GROUP_ACTIVITY`) 일정을 삭제합니다 (soft-delete). 삭제된 일정은 목록/단건 조회에서 노출되지 않습니다. 일정을 삭제해도 해당 일정의 출석 기록(attendance)은 함께 삭제되지 않고 유지됩니다(cascade 없음). 회원 인증이 필요하며, 일반 회원(`USER`)은 본인이 작성한 그룹 활동 일정만 삭제할 수 있고, 운영진(`MANAGER`)도 삭제할 수 있습니다.

> 대상 일정이 그룹 활동(`GROUP_ACTIVITY`)이 아니면 이 엔드포인트로 삭제할 수 없습니다.

## API

```
DELETE /schedules/group-activity/{scheduleId}
```

### 쿼리 파라미터

(해당 없음)

### 요청 바디

(해당 없음)

### 응답 코드 및 응답 바디

```
204 No Content
```

응답 본문 없음.

### 테스트 케이스

1. 정상 삭제 → 204. 이후 목록/단건 조회에서 노출되지 않는다
2. 없는 일정 → 404
3. 대상 일정이 `GROUP_ACTIVITY`가 아니면 → 400 (전용 엔드포인트 사용)
4. 일반 회원(`USER`)이 본인 작성 일정 삭제 → 204
5. 일반 회원이 타인 작성 일정 삭제 시도 → 403
6. 운영진(`MANAGER`)이 본인 작성 일정 삭제 → 204
7. 운영진(`MANAGER`)이 타인 작성 일정 삭제 → 204
8. 인증되지 않은 요청 → 401
9. 일정을 삭제해도 해당 일정의 출석 기록(attendance)은 삭제되지 않고 유지된다
