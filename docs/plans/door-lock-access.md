# 동방 출입 ExPoint 적립

동방 도어락(라즈베리파이)이 사용자의 출입을 백엔드에 보고합니다. 사용자별 KST 달력일자 기준 첫 출입에 대해 시스템 설정값만큼 ExPoint를 적립합니다. 같은 날 두 번째 이후 출입은 적립 없이 정상 응답합니다. 응답으로 출입한 회원의 이름을 반환합니다. SYSTEM API 키 인증을 사용합니다.

## API

```
POST /internal/door-lock/accesses
```

### 요청 헤더

| 이름        |  타입  | 설명             |
| :---------- | :----: | :--------------- |
| `x-api-key` | 문자열 | SYSTEM 인증 키   |

### 쿼리 파라미터

(해당 없음)

### 요청 바디

| 이름         | 타입 | 설명                                        |
| :----------- | :--: | :------------------------------------------ |
| `number`     | 숫자 | 출입한 사용자 학번 (`khlug_members.number`) |
| `roomNumber` | 숫자 | 출입 요청이 발생한 방 번호                  |

### 응답 코드 및 응답 바디

```
200 OK
```

| 이름   |  타입  | 설명             |
| :----- | :----: | :--------------- |
| `name` | 문자열 | 출입한 회원 이름 |

### 테스트 케이스

1. 오늘(KST) 첫 출입이면 ExPoint 적립
   - `x-api-key` 유효, `number`/`roomNumber` 유효, 해당 사용자의 KST 달력일자 기준 첫 호출 → 200, ExPoint 적립, 응답에 회원명(`name`) 포함.
2. 같은 날 두 번째 이후 출입은 적립하지 않음
   - 같은 사용자가 같은 날(KST) 다시 호출 → 200, 적립 없음 (응답에 회원명 포함).
3. `x-api-key` 불일치 → 401
4. `number` 누락 → 400
5. `roomNumber` 누락 → 400
6. 해당 학번을 가진 사용자가 없음 → 404
7. 자정 경계 처리
   - KST 23:59 출입 → 적립. 같은 사용자가 KST 00:01 재출입 → 다시 적립 (다음 달력일자).
