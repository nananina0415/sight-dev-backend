# 일정 출석 등록

운영진이 여러 사용자를 특정 일정에 출석 처리합니다. 출석 처리 시 일정에 설정된 ExPoint(`schedule.expoint`)를 각 사용자에게 적립합니다. 출첵 코드 검증 없이 운영진 권한으로 바로 처리합니다.

## API

```
POST /schedules/{scheduleId}/attendances
```

### 경로 파라미터

| 이름         | 타입 | 설명                     |
| :----------- | :--: | :----------------------- |
| `scheduleId` | 숫자 | 출석 처리할 일정 아이디  |

### 쿼리 파라미터

(해당 없음)

### 요청 바디

| 이름      |   타입    | 설명                  |
| :-------- | :-------: | :-------------------- |
| `userIds` | 숫자 배열 | 출석 처리할 사용자 ID |

### 응답 코드 및 응답 바디

```
201 Created
```

응답 본문 없음.

### 테스트 케이스

1. 운영진이 여러 사용자를 출석 처리할 수 있다
   - 첫 출석인 사용자들에 대해 → 201, attendance row 생성, 각 사용자에게 `schedule.expoint`만큼 ExPoint 적립.
2. 요청 바디의 `userIds`가 빈 리스트면 400
3. 요청 바디에 중복된 `userId`가 포함되면 400
4. 이미 출석 처리된 사용자가 포함되면 409
5. 존재하지 않는 일정 → 404
6. 존재하지 않는 사용자가 포함되면 404
7. 권한 없음 → 403
