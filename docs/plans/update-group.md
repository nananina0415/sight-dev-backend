# 그룹 수정

기존 그룹의 정보를 수정합니다. 변경된 항목에 대해 그룹 멤버 전체에게 알림이 발송됩니다.

## API

```
PUT /groups/{groupId}
```

### 쿼리 파라미터

(해당 없음)

### 요청 바디

| 이름         |  타입  | 설명                                               |
| :----------- | :----: | :------------------------------------------------- |
| `title`      | 문자열 | 그룹 이름                                          |
| `purpose`    | 문자열 | 그룹 목적 (nullable)                               |
| `interest`   | 문자열 | 관심 분야 (`\|`로 구분, nullable)                  |
| `technology` | 문자열 | 기술 스택 (`,`로 구분, nullable)                   |
| `allowJoin`  | 불리언 | 참여 허용 여부                                     |
| `grade`      |  숫자  | 공개 범위 (0=비공개, 2=운영진, 3=회원, 4=완전공개) |
| `repository` | 문자열 | 저장소 URL (nullable)                              |

### 테스트 케이스

1. 그룹장이 그룹 정보를 정상적으로 수정할 수 있다
   - 그룹장이 유효한 수정 요청을 보내면 200 응답과 함께 수정된 그룹 정보를 반환한다
2. 운영 카테고리 그룹은 운영진도 수정할 수 있다
   - 운영 카테고리(`manage`) 그룹에 대해 운영진(그룹장이 아닌)이 수정하면 200 응답을 반환한다
3. 그룹장이 아닌 일반 멤버는 그룹을 수정할 수 없다
   - 그룹장이 아닌 멤버가 수정을 시도하면 403 에러를 반환한다
4. 종료(성공) 상태의 그룹은 수정할 수 없다
   - `end-success` 상태의 그룹을 수정하려 하면 400 에러를 반환한다
5. 종료(실패) 상태의 그룹은 수정할 수 없다
   - `end-fail` 상태의 그룹을 수정하려 하면 400 에러를 반환한다
6. 진행 중 상태의 그룹은 수정할 수 있다
   - `progress` 상태의 그룹을 수정하면 200 응답을 반환한다
7. 중단 상태의 그룹은 수정할 수 있다
   - `suspend` 상태의 그룹을 수정하면 200 응답을 반환한다
8. 운영진 공개(grade=2)는 운영진만 설정할 수 있다
   - 일반 회원(그룹장)이 grade를 2로 변경하려 하면 403 에러를 반환한다
9. 그룹 수정 시 변경된 항목에 대해 멤버 전체에게 알림이 발송된다
   - 그룹 정보를 수정하면 그룹 멤버 전체에게 알림이 발송된다
10. 그룹 수정은 활동 갱신(`changed_at`)에 영향을 주지 않는다
    - 그룹 정보를 수정해도 `changed_at`은 갱신되지 않는다 (자동 중단 판정에 영향 없음)
