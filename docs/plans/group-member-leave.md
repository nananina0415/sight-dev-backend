# 그룹 탈퇴

그룹에서 탈퇴합니다.

## API

```
DELETE /groups/{groupId}/members/@me
```

### 쿼리 파라미터

(해당 없음)

### 요청 바디

(해당 없음)

### 응답 코드 및 응답 바디

```
204 No Content
```

### 테스트 케이스

1. 그룹 멤버가 정상적으로 탈퇴할 수 있다
   - `group_member` 레코드가 삭제되고 `group.count_member`가 1 감소한다
   - `group.changed_at`이 갱신된다. 중단 상태(`suspend`)인 그룹이면 진행 중(`progress`)으로 자동 전환된다
   - 탈퇴자에게 -10 ExPoint가 차감된다. 활용 실습 그룹(`id=7549`)은 차감하지 않는다
   - 운영 카테고리(`category=manage`) 그룹이면 전 멤버에게 알림이 발송된다. 그 외엔 탈퇴자 본인과 그룹장에게 알림이 발송된다

2. 운영 카테고리가 아닌 그룹에서 그룹장은 탈퇴할 수 없다 — 400
   - 먼저 다른 멤버에게 그룹장을 위임해야 한다
   - 운영 카테고리 그룹에서의 그룹장 탈퇴는 허용되지만, 탈퇴 후 `group.master`가 가리키는 처리 방식은 기존 구현 확인 필요 (결정 보류)

3. 그룹 멤버가 아니면 400을 반환한다

4. 존재하지 않는 그룹이면 404를 반환한다
